# README.md Writing Guide

## Structure Checklist

```
[ ] Center-aligned logo/banner + project name
[ ] Badge row (build status, version, license, etc.)
[ ] One-line description (explain "what it does" within 5 seconds)
[ ] Screenshot / demo GIF (if available)
[ ] Table of Contents (only for 300+ line READMEs)
[ ] Features (bullet list)
[ ] Quick Start / Installation
[ ] Usage examples (code blocks required)
[ ] Architecture diagram (prefer Mermaid over ASCII art)
[ ] API reference (use details/summary for response examples)
[ ] Environment variables table
[ ] Contributing guide / Related links
[ ] License
```

## Things to Avoid
- Broken badge URLs (always replace USER/REPO with actual values)
- Installation code examples that don't work
- Outdated screenshots
- Unnecessarily verbose descriptions
- Missing license
